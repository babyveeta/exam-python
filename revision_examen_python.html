Somme pondérée récursive
Fonction récursive — sommep
Énoncé

On vous demande de définir une fonction nommée sommep qui accepte en argument une liste pouvant récursivement contenir d'autres listes, et qui retourne la somme pondérée des nombres qui s'y trouvent.

La somme pondérée d'une liste [a₁, a₂, …, aₙ] se calcule par la formule :

∑ i × aᵢ (pour i allant de 1 à n)

Par exemple, l'expression sommep([2, 7, 1, 3]) doit retourner la valeur :

1×2 + 2×7 + 3×1 + 4×3 = 31

Lorsqu'un élément de la liste est une autre liste, il s'agit simplement de faire un appel récursif sur la liste. Par exemple, l'expression sommep([7, [3, 1], 5]) doit retourner la valeur :

1×7 + 2×(1×3 + 2×1) + 3×5 = 32

Votre fonction doit accepter un nombre arbitraire de listes imbriquées.

Bref, lorsqu'un élément est un entier, il faut ajouter à la somme son rang fois sa valeur, et lorsque l'élément est une liste, il faut ajouter à la somme son rang fois la valeur d'un appel récursif sur la liste en question. Tous les types de données autres que les entiers et les listes doivent être ignorés (on passe au suivant).

Notez que pour cet exercice, les rangs débutent toujours à 1.

Rappel : pour déterminer si un objet est bien une instance d'une classe w, faites simplement appel à la fonction isinstance.
Solution de référence
  sommep.py
def sommep(liste):
    # initialiser une variable d'accumulation
    somme = 0
    for i, val in enumerate(liste, 1):
        if isinstance(val, int):
            somme += i*val

        elif isinstance(val, list):
            somme += i*sommep(val)

    return somme

Classe composée
Classe Python — AetB
Énoncé

Soit les classes A et B définies dans la cellule de contexte. Il s'agit de deux classes dont les constructeurs acceptent un argument pouvant être soit un nombre entier, soit une chaîne de caractères. Étudiez ces deux classes afin de bien comprendre leur fonctionnement.

On vous demande de définir une classe nommée AetB qui est composée d'une instance de la classe A et d'une instance de la classe B. Votre classe doit définir les cinq (5) méthodes suivantes :

  1. Un constructeur qui accepte deux arguments, chacun pouvant être soit un nombre, soit une chaîne de caractères. À partir de ces deux arguments, votre constructeur doit initialiser deux variables d'instance nommées A et B (attention à la casse majuscule), la 1re en créant une instance de la classe A, et la 2e en créant une instance de la classe B.
  2. Une méthode spéciale de conversion en chaîne de caractères.
  3. Une méthode spéciale pour déterminer si deux instances sont équivalentes (opérateur ==) ; deux instances AetB sont équivalentes si et seulement si leurs deux composantes sont équivalentes.
  4. Une méthode spéciale pour implanter la somme de deux instances (opérateur +).
  5. Une méthode spéciale pour implanter l'opérateur [] afin d'extraire l'une ou l'autre des deux composantes de l'instance. L'indice 0 sera utilisé pour désigner la composante A, alors que l'indice 1 désignera la composante B. Tout autre indice doit soulever une exception de type IndexError.

Par exemple, l'expression print(AetB(1, 2)) doit afficher :

(A: 1) et (B: 2)

L'expression AetB(1, 2) + AetB(3, 4) doit retourner une instance AetB(1+3, 2+4) (ce qui équivaut à AetB(4, 6)).

L'expression AetB(1, 2)[0] doit retourner A(1) et AetB(1, 2)[1] doit retourner B(2), et AetB(1, 2)[2] doit soulever une exception de type IndexError.

Tests de validation

assert AetB(1, 2).B == B(2)

Solution de référence
  AetB.py
class AetB:
    def __init__(self, a, b):
        self.A = A(a)
        self.B = B(b)

    def __str__(self):
        return f'({self.A}) et ({self.B})'

    def __eq__(self, other):
        return self.A == other.A and self.B == other.B

    def __add__(self, other):
        return AetB(self.A.a+other.A.a, self.B.b+other.B.b)

    def __getitem__(self, indice):
        if not 0 <= indice <= 1:
            raise IndexError(f"Votre indice {indice} est invalide")
        return self.A if indice == 0 else self.B

Décorateur d'authentification
Décorateur paramétré — authentification
Énoncé

Votre tâche est de créer un décorateur sous forme de fonction nommé authentification qui permettra d'ajouter des informations d'authentification à une session avant d'appeler une fonction. Le décorateur doit prendre deux paramètres, idul et secret, et doit ajouter un attribut auth à la session avec ces informations. Voici un exemple d'utilisation du décorateur :

@authentification("josmi42", "2b0caf03-498f-4d30-8e6d-af3512d66cf5")
def obtenir_données(session, url, params):
    return session.get(url, params=params)

Dans cet exemple, la fonction obtenir_données est décorée avec le décorateur authentification. Lorsque la fonction est appelée, le décorateur ajoute les informations d'authentification à la session avant d'appeler la fonction. Vous devez implémenter le décorateur authentification de manière à ce qu'il fonctionne correctement avec la fonction fournie.

L'ajout des informations d'authentification à la session se fait en modifiant l'attribut auth de la session :

session.auth = (idul, secret)

Notez que la fonction décorée peut prendre des paramètres arbitraires en plus de la session, mais que le premier paramètre sera toujours l'argument session. Par exemple, en ajoutant un paramètre data à la fonction :

@authentification("josmi42", "2b0caf03-498f-4d30-8e6d-af3512d66cf5")
def envoyer_données(session, url, params, data):
    return session.post(url, params=params, data=data)

Pour qu'un décorateur sous forme de fonction puisse accepter des paramètres, il doit être défini comme une fonction, ayant comme paramètres ceux souhaités, et qui retourne une fonction décorateur tel que montré dans le module optionnel touchant aux décorateurs. Cela implique donc de définir trois (3) fonctions imbriquées.

Solution de référence
  authentification.py
# 1re fonction qui accepte les paramètres du décorateur
def authentification(idul, secret):
    # 2e fonction qui accepte la fonction à décorer
    def decorator(function):
        # 3e fonction qui enrobe la fonction décorée
        def wrapper(session, *args, **kwargs):
            # ajouter l'authentification à la session
            session.auth = (idul, secret)

            # retourner le résultat
            return function(session, *args, **kwargs)

        # retourner la fonction décorée
        return wrapper

    # retourner le décorateur paramétré
    return decorator

Décorateur horodateur
Décorateur simple — horodateur
Énoncé

Définissez un décorateur nommé horodateur qui, à chaque appel de la fonction décorée, affiche la date de l'appel ainsi que la valeur de ses arguments. Implantez votre décorateur sous la forme d'une fonction qui accepte en argument une autre fonction.

Sur une même ligne, le format d'affichage doit comprendre les éléments suivants :

  1. la date du jour sous le format aaaa-mm-jj (le format par défaut des dates) ;
  2. un espace ;
  3. le nom de la fonction décorée ;
  4. une parenthèse ouvrante ;
  5. les arguments positionnels utilisés lors de l'appel, séparés par des virgules ;
  6. les arguments nommés utilisés lors de l'appel, séparés par des virgules ;
  7. une parenthèse fermante.

Par exemple :

@horodateur
def fonction_de_test(*args, **kargs):
    pass

appelée avec les arguments :

fonction_de_test(1, 5, 3, a=9, z=13)

doit produire l'affichage :

2025-04-24 fonction_de_test(1, 5, 3, a=9, z=13)

Notez bien que ceci n'est qu'un exemple pour un appel effectué le 24 avril 2025. Votre décorateur doit être général, c'est-à-dire produire l'affichage qui correspond à la date du jour, au nom de la fonction décorée ainsi qu'aux valeurs effectives des arguments utilisés lors de l'appel. N'oubliez pas non plus que votre décorateur doit appeler la fonction décorée avec l'ensemble de ses arguments (positionnels et nommés) et retourner son résultat.

Dans le cas des arguments nommés, assurez-vous de les énumérer dans l'ordre lexicographique (l'ordre des mots d'un dictionnaire traditionnel).

Indices :
  1. pour obtenir la date du jour, faites appel à la méthode today de la classe datetime.date (cette classe est d'ailleurs déjà importée dans le contexte de cet exercice) ;
  2. pour connaître le nom d'un objet de la classe function, il suffit d'accéder à son attribut __name__ ;
  3. affichez les valeurs des arguments reçus en utilisant repr pour les convertir en chaînes de caractères, et utilisez la méthode join pour produire des séquences de chaînes séparées par des virgules ;
  4. pour le tri lexicographique, utilisez simplement sorted.
Contexte de l'exercice : from datetime import date
Solution de référence
  horodateur.py
from datetime import date

def horodateur(function):
    def wrapper(*args, **kwargs):
        # construire la liste des arguments positionnels
        positionnels = [repr(a) for a in args]

        # construire la liste des arguments nommés, triés lexicographiquement
        nommes = [f"{k}={repr(v)}" for k, v in sorted(kwargs.items())]

        # afficher date + nom + arguments
        print(f"{date.today()} {function.__name__}({', '.join(positionnels + nommes)})")

        # appeler et retourner le résultat
        return function(*args, **kwargs)

    return wrapper

Jeu de la vie
Automate cellulaire — jeu_de_vie
Énoncé

Le jeu de la vie est un jeu de simulation mathématique sans joueur explicite, où des cellules vivantes évoluent de génération en génération. Il s'agit d'un automate qui joue de lui-même. Le jeu est habituellement représenté sur une grille 2D dont les dimensions sont a priori indéfinies. Chaque case de la grille peut héberger une cellule, auquel cas on dit que la case correspondante est vivante. Sinon, la case est morte. L'état d'une case à la génération suivante dépend de son état actuel ainsi que de l'état de ses huit cases adjacentes.

Les règles du jeu sont les suivantes :

  1. Toute case vivante entourée d'exactement deux (2) ou trois (3) cases vivantes va survivre à la génération suivante.
  2. Toute case morte entourée d'exactement trois (3) cases vivantes va renaître à la génération suivante.
  3. Dans tous les autres cas, la case sera morte à la génération suivante.

Par exemple, voici une grille 6×6 qui selon ces règles produira une oscillation de génération en génération :

. . . . . . . . . . . . . . . . . . . . . . . . . . + . . . . . . . . . . . + + + . → . + . + . . → . . + + + . → etc. . + + + . . . + . + . . . + + + . . . . . . . . . . + . . . . . . . . . . . . . . . . . . . . . . . . . . .

où les symboles + et . représentent respectivement les cases vivantes et mortes.

On vous demande de coder une fonction nommée jeu_de_vie qui accepte en entrée un ensemble de tuples (i, j) représentant les coordonnées des cases vivantes actuelles, et qui retourne en sortie l'ensemble des cases vivantes pour la génération suivante dans la séquence du jeu.

Indices pour résoudre ce problème :
  1. Initialiser un dictionnaire vide afin de pouvoir compter les nombres de voisins de chaque case potentielle.
  2. Pour chaque case (i, j) de l'ensemble actuel des cases vivantes :
    • A. Créez la liste des huit (8) cases voisines.
    • B. Pour chacune de ces cases voisines : Ajoutez un (+1) à la case correspondante dans un dictionnaire de voisinage.
  3. Parcourir les éléments du dictionnaire de voisinage en ne retenant que les cases qui respectent les règles du jeu quant au nombre de voisins.
  4. Retourner l'ensemble de ces cases vivantes pour la génération suivante du jeu.
Note : dans le contexte d'un énoncé if, l'opérateur in peut servir à tester si une clé appartient à un dictionnaire ou à un ensemble. Dans le contexte d'un énoncé for, ce même opérateur permet d'itérer sur les clés d'un dictionnaire, ou sur les éléments d'un ensemble.
Tests de validation
for config in [tob, blinker, toad, beacon]:
    print(jeu_de_vie(config))

{(1, 0), (1, 1), (1, 2)}
{(3, 1), (2, 0), (2, 3), (0, 2), (1, 0), (1, 3)}
{(0, 1), (0, 0), (2, 3), (3, 3), (1, 0), (3, 2)}
Notez que l'ordre des éléments d'un ensemble est imprévisible et sans importance.
Solution de référence
  jeu_de_vie.py
def jeu_de_vie(cases):
    # initialiser le dictionnaire de voisinage
    voisinage = {}

    # pour chaque case vivante
    for case in cases:
        i, j = case
        # créer la liste des huit cases voisines
        voisins = [
            (i-1, j-1),
            (i,   j-1),
            (i+1, j-1),
            (i-1, j),
            (i+1, j),
            (i-1, j+1),
            (i,   j+1),
            (i+1, j+1)
        ]

        # ajouter un (1) à chaque voisin
        for voisin in voisins:
            voisinage[voisin] = voisinage.get(voisin, 0) + 1

    # retourner les cases du voisinage qui respectent les règles du jeu
    return {
        case for case in voisinage
        if case in cases and 2 <= voisinage[case] <= 3
        or voisinage[case] == 3
    }

Grille en art ASCII
Fonction — grille_ascii
Énoncé

Codez une fonction nommée grille_ascii qui accepte en entrée trois (3) arguments :

  1. Le nombre m > 0 de lignes de la grille ;
  2. Le nombre n > 0 de colonnes de la grille ;
  3. L'ensemble des tuples (i, j) des cases actives de la grille.

Pour les coordonnées des cases de la grille, nous adopterons la convention habituelle du Python, à savoir que les indices débutent à zéro, et nous placerons l'origine de la grille en haut à gauche.

Votre fonction doit retourner une chaîne de caractères qui représente la grille en « art ASCII » selon le format suivant. Les cases actives y sont représentées par des plus (+) et les autres cases par des points (.). Par exemple, l'expression suivante :

grille_ascii(3, 3, {(0, 1), (1, 0), (1, 2), (2, 1)})

doit produire la chaîne suivante :

' -----\n|. + .|\n|+ . +|\n|. + .|\n -----'

qui lorsqu'affichée avec print produira :

----- |. + .| |+ . +| |. + .| -----

Notez qu'un espace est inséré entre les cases d'une même ligne afin d'équilibrer les espacements horizontal et vertical.

Testez bien votre solution avec les tuples d'arguments définis dans le contexte de cet exercice. Par exemple, en exécutant :

configurations = [tob, blinker, toad, beacon]
for config in configurations:
    print(grille_ascii(*config))

Vous devez obtenir à la console :

----- |. + .| |+ . +| |. + .| ----- ----- |. + .| |. + .| |. + .| ----- ------- |. . . .| |. + + +| |+ + + .| |. . . .| ------- ------- |+ + . .| |+ + . .| |. . + +| |. . + +| -------
Solution de référence
  grille_ascii.py
def grille_ascii(m, n, cases):
    # créer l'en-tête de la grille
    en_tête = f" {'-'*(2*n-1)}"

    # insérer l'en-tête devant la première ligne
    lignes = [en_tête]

    # pour chaque ligne
    for i in range(m):
        ligne_courante = []
        # et chaque colonne
        for j in range(n):
            if (i, j) in cases:
                # la case est active
                ligne_courante.append('+')
            else:
                # la case est inactive
                ligne_courante.append('.')

        # joindre les colonnes et ajouter une nouvelle ligne
        lignes.append(f"|{' '.join(ligne_courante)}|")

    # insérer l'en-tête à la suite de la dernière ligne
    lignes.append(en_tête)

    # retourner la jointure des différentes lignes
    return '\n'.join(lignes)

Vecteur 3D
Classe Python — Vecteur3D
Énoncé

Créez une classe Vecteur3D qui encapsule un vecteur à trois dimensions. La classe doit définir les méthodes suivantes :

  • Un constructeur __init__(self, x, y, z) qui initialise les trois composantes du vecteur.
  • L'opérateur addition + (méthode __add__) entre deux Vecteur3D.
  • L'opérateur soustraction - (méthode __sub__) entre deux Vecteur3D.
  • L'opérateur multiplication * (méthode __mul__) : soit entre deux Vecteur3D (produit composante par composante), soit entre un Vecteur3D et un scalaire (int ou float).
  • La conversion en chaîne de caractères __str__ qui retourne (x, y, z).
  • La méthode norme qui calcule la norme euclidienne du vecteur.

Dans votre constructeur, assurez-vous que les trois arguments reçus sont bien des nombres, c'est-à-dire des instances des classes int ou float. De même, pour vos opérateurs +, - et *, assurez-vous que l'opérande à la droite de l'opérateur est valide. Sinon, soulevez une exception de type TypeError.

Rappel : pour déterminer si un objet object est bien une instance de la classe w, faites simplement appel à la fonction isinstance.

Par exemple :

x = Vecteur3D(1, 2, 3)
y = Vecteur3D(4, 5, 6)
print('x =', x)
print('y =', y)
print('x+y =', x+y)
print('x-y =', x-y)
print('x*y =', x*y)
print('x*2 =', x*2)
print('x.norme() =', x.norme())

Doit produire :

x = (1, 2, 3) y = (4, 5, 6) x+y = (5, 7, 9) x-y = (-3, -3, -3) x*y = (4, 10, 18) x*2 = (2, 4, 6) x.norme() = 3.7416573867739413
Solution de référence
  Vecteur3D.py
class Vecteur3D:
    '''Classe qui encapsule un vecteur à trois dimensions.'''
    def __init__(self, x, y, z):
        '''Construire un vecteur.'''
        if not isinstance(x, (int, float)):
            raise TypeError(f"Votre composante x de type {type(x)} est invalide.")
        self.x = x

        if not isinstance(y, (int, float)):
            raise TypeError(f"Votre composante y de type {type(y)} est invalide.")
        self.y = y

        if not isinstance(z, (int, float)):
            raise TypeError(f"Votre composante z de type {type(z)} est invalide.")
        self.z = z

    def __add__(self, other):
        '''Additionner deux vecteurs.'''
        if not isinstance(other, Vecteur3D):
            raise TypeError(f"Le type {type(other)} est invalide pour l'addition")
        return Vecteur3D(self.x+other.x, self.y+other.y, self.z+other.z)

    def __sub__(self, other):
        '''Soustraire deux vecteurs.'''
        if not isinstance(other, Vecteur3D):
            raise TypeError(f"Le type {type(other)} est invalide pour la soustraction")
        return Vecteur3D(self.x-other.x, self.y-other.y, self.z-other.z)

    def __mul__(self, other):
        '''Multiplier deux vecteurs, ou un vecteur par un scalaire.'''
        if isinstance(other, Vecteur3D):
            return Vecteur3D(self.x*other.x, self.y*other.y, self.z*other.z)
        elif isinstance(other, (int, float)):
            return Vecteur3D(self.x*other, self.y*other, self.z*other)
        else:
            raise TypeError(f"Le type {type(other)} est invalide pour la multiplication")

    def __str__(self):
        '''Convertir le vecteur en chaîne de caractères.'''
        return f'({self.x}, {self.y}, {self.z})'

    def norme(self):
        '''Calculer la norme du vecteur.'''
        return (self.x**2 + self.y**2 + self.z**2)**.5

Ma liste particulière
Héritage de list — MaListe
Énoncé

On vous demande de définir une classe nommée MaListe qui hérite de toutes les fonctionnalités d'une liste standard (classe list), mais avec les particularités suivantes :

  • Une méthode spéciale pour l'addition de deux listes. On veut que cette méthode retourne une nouvelle instance de MaListe dont chaque élément est le résultat de la somme des éléments correspondants des deux listes. Par exemple, MaListe([1, 2, 3]) + MaListe([4, 5, 6]) doit retourner MaListe([5, 7, 9]). Si les deux listes de départ n'ont pas la même longueur, votre fonction doit soulever une exception de type ValueError.
  • Une méthode nommée somme qui retourne la somme des éléments de la liste.
  • Une méthode nommée moyenne qui retourne la moyenne des éléments de la liste (la division de la somme par la longueur). Cette méthode devrait faire appel à la méthode somme pour calculer la somme et à la fonction len pour déterminer la longueur de la liste.
ATTENTION : assurez-vous d'utiliser la liste dont vous héritez et non d'ajouter une liste en variable d'instance.

Vous pouvez supposer que tous les éléments qui seront ajoutés à votre liste sont des entiers.

Solution de référence
  MaListe.py
class MaListe(list):
    def __add__(self, other):
        # s'assurer que les deux listes ont la même longueur
        if len(self) != len(other):
            raise ValueError("Vos deux listes n'ont pas la même longueur")

        return MaListe(x+y for x, y in zip(self, other))

    def somme(self):
        # retourner la somme des éléments
        return sum(self)

    def moyenne(self):
        # retourner la somme divisée par la longueur
        return self.somme() / len(self)

Paquet de cartes
Héritage de list — PaquetDeCartes
Contexte — Classe CarteBase

class CarteBase:
    noms   = {1: 'as', 11: 'valet', 12: 'dame', 13: 'roi'}
    sortes = ['pique', 'cœur', 'carreau', 'trèfle']

    def __init__(self, m, n):
        assert 1 <= m <= 13 and 1 <= n <= 4
        self.numface  = m
        self.numsorte = n

    def __repr__(self):
        return f'{type(self).__name__}({self.numface}, {self.numsorte})'

    def __str__(self):
        return f'{self.nom()} de {self.sorte()}'

    def nom(self):
        # retourne le nom de la carte (as, valet, dame, roi) ou le numéro
        return CarteBase.noms.get(self.numface, str(self.numface))

    def sorte(self):
        # retourne la sorte de carte (pique, cœur, carreau ou trèfle)
        return CarteBase.sortes[self.numsorte-1]

    def force(self):
        # retourner le numéro de la face, sauf pour l'as qui vaut plus que le roi
        return self.numface if self.numface > 1 else 14
Énoncé

Créez une classe nommée PaquetDeCartes qui encapsule un paquet de 52 cartes de jeu. Faites hériter votre classe de la classe standard list en vous assurant d'utiliser cette liste dont vous héritez afin de contenir votre séquence de cartes. Ne pas créer d'autres instance de liste.

Outre l'interface d'une liste, votre classe doit posséder les méthodes suivantes :

  1. __init__(self, Carte, mélanger=False) qui permet de construire un paquet de 52 cartes en instanciant des cartes à partir d'une classe Carte reçue en argument. import random

class PaquetDeCartes(list):
    def __init__(self, carte, mélanger=False):
        # initialiser la liste
        super().__init__()

        # créer toutes les cartes dans l'ordre
        for couleur in range(1, 5):
            for face in range(1, 14):
                self.append(carte(face, couleur))

        if mélanger:
            # mélanger le paquet tel que spécifié
            self.mélanger()

    def mélanger(self):
        # mélanger aléatoirement les cartes
        random.shuffle(self)

    def piger(self, n):
        # initialiser la pige
        pige = []
        for i in range(n):
            if self:
                # ajouter la première carte à la pige
                pige.append(self.pop(0))

        # retourner la pige
        return pige

Énoncé

Soit la classe CarteBase présente dans le contexte de cet exercice (voir Exercice 9). Elle encapsule une carte de jeu caractérisée par des valeurs numériques pour sa face et sa sorte. Les faces possibles sont des entiers de 1 à 13, avec la face 1 correspondant à l'as, et les faces 11, 12 et 13 correspondant respectivement au valet, à la dame et au roi. Les autres faces sont simplement identifiées par leur numéro de 2 à 10. Les sortes possibles sont des entiers de 1 à 4 correspondants respectivement aux cartes de pique, cœur, carreau et trèfle. Analysez cette classe afin de bien comprendre son fonctionnement.

  3. __ge__(self, autre) qui permet de définir le comportement de l'opérateur >=.
